Enhanced Iframe Integration

ForumAI provides powerful iframe embedding options that allow you to integrate your forum directly into your website with enhanced cross-domain communication, responsive design, and event tracking capabilities.

Note: The iframe integration method is recommended for most use cases as it provides the best user experience and integrates seamlessly with your website.

Iframe Parameters

The ForumAI iframe embedding system supports a wide range of parameters to customize the appearance and behavior of your embedded forum:

Parameter Type Default Description
forumId Number Required The ID of your forum
showHeader Boolean true Whether to show the forum header
showFooter Boolean true Whether to show the "Powered by ForumAI" footer
theme String "light" Theme to use ("light", "dark", or "auto")
initialView String "questions" Initial view to show ("questions", "categories", "search")
defaultHeight Number 600 Default iframe height in pixels
maxHeight Number 1200 Maximum iframe height in pixels
width String "100%" Width of the iframe (CSS value)
autoResize Boolean true Whether to automatically resize the iframe based on content
questionId Number null ID of a specific question to display
categoryId Number null ID of a specific category to display
crossDomain String "secure" Cross-domain messaging policy ("secure", "allow", or "block")
trackEvents Boolean true Whether to track events for analytics
loadCallback String null Name of a global function to call when the iframe is loaded

Basic Iframe Implementation Example

<div id="forumAI-container"></div>

<script src="https://formai.repl.app/embed/forum-embed.js"></script>
<script>
  window.ForumAIEmbedConfig = {
    container: "forumAI-container",
    forumId: 123,
    showHeader: true,
    autoResize: true,
    crossDomain: "allow"
  };
</script>

Enhanced Redirect Options

ForumAI provides advanced redirect options that allow you to integrate with your forum through server-side redirects, custom landing pages, and specialized user experiences.

New Feature: The enhanced redirect system now supports multiple redirect modes, UTM tracking parameters, and custom behaviors.

Redirect Parameters

The redirect endpoint supports numerous parameters to control the destination and behavior of the redirect:

Parameter Type Default Description
dest String "forum" Destination type: "forum", "question", "category", "ask"
id Number null ID of the question or category when dest is "question" or "category"
embed String null Embed mode: "true", "minimal", "compact", "sidebar"
theme String null Theme to use: "light", "dark", "auto"
mode String "navigate" Redirect mode: "navigate", "newtab", "iframe"
utm_source String "redirect" UTM source parameter for tracking
utm_medium String "forum" UTM medium parameter for tracking
utm_campaign String "forum-redirect" UTM campaign parameter for tracking
track Boolean true Whether to track the redirect ("true" or "false")
cookies Boolean false Whether cookies are allowed ("true" or "false")
data Boolean false Whether data collection is allowed ("true" or "false")
params Boolean false Whether to preserve additional query params ("true" or "false")
custom JSON null JSON string of custom parameters to pass to the forum

Redirect Modes

The redirect endpoint supports three different modes:

  • navigate - Standard redirect that immediately navigates the user to the forum
  • newtab - Shows a landing page with a button that opens the forum in a new tab
  • iframe - Embeds the forum in an iframe on a landing page

Redirect URL Examples

// Basic redirect to forum homepage
https://formai.repl.app/embed/redirect/123

// Redirect to a specific question with tracking
https://formai.repl.app/embed/redirect/123?dest=question&id=456&utm_source=blog&utm_medium=link

// Redirect with a landing page that opens in a new tab
https://formai.repl.app/embed/redirect/123?mode=newtab&dest=ask

// Embed the forum in an iframe
https://formai.repl.app/embed/redirect/123?mode=iframe&theme=dark

Communication Library

The ForumAI Communication Library provides a robust API for communicating with embedded forums, handling events, and controlling forum behavior from your parent website.

New Feature: The communication library offers a comprehensive set of methods for managing bidirectional communication between your website and the embedded forum.

Setup

<script src="https://formai.repl.app/embed/forum-communication.js"></script>
<script>
  // Initialize the communication API
  const forumCommunication = ForumAI.initCommunication({
    embedId: "forumAI-container",
    debug: true
  });
</script>

Library Methods

The communication library provides the following methods:

Method Parameters Description
navigate(path) path: String Navigate to a specific path in the forum
refresh() None Refresh the forum content
scrollTo(selector) selector: String Scroll the forum to a specific element
setTheme(theme) theme: String Set the theme for the forum
search(query) query: String Search the forum
sendMessage(type, data) type: String, data: Object Send a custom message to the forum
on(eventType, callback) eventType: String, callback: Function Add an event listener
off(eventType, callback) eventType: String, callback: Function Remove an event listener
getState() None Get the current state of the forum
isLoaded() None Check if the forum is loaded
getCurrentPath() None Get the current path of the forum
getViewMode() None Get the current view mode

Usage Examples

// Navigate to the ask page
forumCommunication.navigate('/ask');

// Change the theme
forumCommunication.setTheme('dark');

// Get the current state
const state = forumCommunication.getState();
console.log('Current forum state:', state);

// Listen for navigation events
forumCommunication.on('navigate', (detail) => {
  console.log('Forum navigated to:', detail.path);
});

Event Handling

The communication library emits events that you can listen for to respond to actions in the embedded forum:

Event Types

Event Type Description Data Properties
resize Fired when the iframe is resized height, embedId
navigate Fired when navigation occurs in the forum path, title, embedId
track Fired for tracking events action, category, label, value, embedId
load Fired when the forum is fully loaded embedId
interaction Fired when users interact with forum elements action, elementType, elementId, embedId
custom Fired for custom events from the forum Custom properties and embedId
stateChange Fired when forum state changes type, oldValue, newValue, state

Event Handling Examples

// Listen for forum load
forumCommunication.on('load', () => {
  console.log('Forum loaded!');
});

// Track user interactions
forumCommunication.on('interaction', (detail) => {
  if (detail.action === 'like' || detail.action === 'answer') {
    console.log(`User ${detail.action}d content:`, detail);
  }
});

// Respond to navigation
forumCommunication.on('navigate', (detail) => {
  // Update breadcrumbs or analytics
  updateBreadcrumbs(detail.path, detail.title);
});

// Handle custom events
forumCommunication.on('custom', (detail) => {
  if (detail.key === 'lead_capture') {
    showThankYouMessage(detail.email);
  }
});

Customization

ForumAI provides extensive customization options to match the look and feel of your website:

Styling Options

You can customize the appearance of the embedded forum through the theme color and styling options in your forum settings. Additionally, you can use CSS variables to override the default styles: 

<style>
  #forumAI-container {
    --formai-primary-color: #5e35b1;
    --formai-secondary-color: #9575cd;
    --formai-background: #f5f5f7;
    --formai-text-color: #333;
    --formai-border-radius: 8px;
    --formai-font-family: 'Roboto', sans-serif;
    --formai-header-bg: linear-gradient(45deg, #5e35b1, #3949ab);
  }
</style>

Callback Configuration

You can configure callbacks to respond to specific events: 

const forumCommunication = ForumAI.initCommunication({
  embedId: "forumAI-container",
  debug: true,
  onLoad: (detail) => {
    console.log('Forum loaded!');
    hideLoadingSpinner();
  },
  onNavigate: (detail) => {
    updateBreadcrumbs(detail.path, detail.title);
  },
  navigationCallbacks: {
    // Called when user visits a question page
    question: (detail) => {
      logQuestionView(detail.path.split('/')[2]); // Extract question ID
    },
    // Called when user visits category page
    category: (detail) => {
      updateCategoryMetrics(detail);
    }
  },
  trackingCallbacks: {
    // Called when a view event is tracked
    'forum:view': (detail) => {
      incrementPageViews();
    }
  }
});

Best Practices

Follow these best practices for optimal forum integration:

Performance Optimization

  • Load the forum embed script asynchronously to prevent blocking page rendering
  • Set appropriate defaultHeight and maxHeight values to minimize layout shifts
  • Use the autoResize parameter to ensure the forum fits its content
  • Consider lazy-loading the forum when it comes into viewport for below-the-fold placements

Security Considerations

Important: When using crossDomain: "allow", ensure you understand the security implications. Only enable this for domains you trust.
  • Use crossDomain: "secure" (default) to only allow communication with trusted domains
  • Implement proper message validation when handling events from the iframe
  • Use HTTPS for both your website and forum to ensure secure communication

User Experience Recommendations

  • Clearly indicate to users when they are interacting with an embedded forum
  • Customize the forum's appearance to match your website's design
  • Implement smooth transitions when navigating within the forum
  • Consider using the navigation event to update browser history for better back/forward navigation
  • Test the forum integration on mobile devices to ensure responsiveness

SEO Considerations

  • Use the redirect system for pages where SEO is important, as content in iframes isn't indexable
  • Include relevant keywords and descriptions in your page content around the embedded forum
  • Consider using the newtab mode for forums with rich SEO content
